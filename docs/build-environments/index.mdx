Ready to let any agent control your software? hud environments wrap apps, games, APIs – anything Docker-able – in the Model Context Protocol (MCP).

This page is a **hands-on quick-start**. Follow it end-to-end, then jump to advanced docs when you need more.

## 1 · Create a project (1 min)

```bash
# Pick a template: blank, deep-research, browser
hud init my-env
```

`hud init` drops in the latest template from the SDK:

```
my-env/
├── Dockerfile
├── pyproject.toml          # deps & metadata
├── controller/             # MCP frontend (stateless)
│   ├── __init__.py         # mcp = MCPServer()
│   ├── __main__.py         # python -m controller → mcp.run()
│   ├── hooks.py            # @mcp.initialize / @mcp.shutdown
│   └── tools.py            # @mcp.tool act / setup / evaluate
└── environment/            # FastAPI backend (stateful)
    └── server.py           # /health /act /reset /state
```

## 2 · Run the dev loop (2 min)

```bash
# Build once, then hot-reload forever
hud dev --build --interactive
```

* `hud dev` mounts your source, restarts the controller on save, and leaves backend state intact.
* Use ↑ ↓ ↵ to call tools live.

### Edit-save-test flow
1. Open `controller/tools.py`, add or tweak a tool.
2. Save – the controller restarts instantly.
3. In the interactive UI press ↵ to re-run the tool.

## 3 · Debug if necessary

If the controller fails to start or tools disappear, run the 5-phase checker:

```bash
hud debug my-env:dev   # deep compliance check – only when something is broken
```

and follow the hints it prints.

## 4 · Build & push (1 min)

```bash
hud build         # creates hud.lock.yaml & production image
hud push          # push to registry (needs HUD_API_KEY & docker login)
```

After a successful push, the lock file records the unique immutable digest.

## 5 · Use it in RL / evals (optional)

```bash
# Convert local tasks → remote and launch RL flow
hud rl tasks.json
```

The CLI will build / push if you forgot, then run training against the cloud service.

| Action | Command |
|--------|---------|
| Create env | `hud init my-env -p blank` |

| Troubleshoot | `hud debug my-env:dev` |
| Build image | `hud build` |
| Push to registry | `hud push` |
| RL training | `hud rl tasks.json` |
